git同时配置github和gitee

创建多个ssh key

  • 一个github账号可以有多个设备的 ssh-key 与其关联。在不同的设备上使用同一个账号，生成不同的ssh-key，然后将由该github账号在不同电脑上生成的的ssh-key进行关联。

    一个账号在不同的设备上都可以生成一个ssh-key，即一个github账号可以与多个设备进行关联

  • 一个设备也可以使用多个github账号生成多个ssh-key。一个设备可以使用多个github账号，生成多个ssh-key，将每个账号生成的ssh-key与对应的github账号进行关联。

    即一个设备可以生成多个ssh-key与多个github账号进行关联。

对于我来说我有一个gitee账号，和一个github账号，也有两台设备：一个台式，一个笔记本，所以创建多个ssh-key是十分有必要的。

查看已经存在的ssh key

打开git bash，输入：ll ~/.ssh，查看已经生成的ssh-key

生成ssh-key

生成一个 Github 用的 SSH-Key，其中 github_id_rsa 为密钥的文件名，~/.ssh/github_id_rsa 为密钥目录位置：

使用命令: ssh-keygen -t rsa -C '[邮箱]' -f ~/.ssh/github_id_rsa,然后按三次enter即可

  • 邮箱使用自己的github账号即可，邮箱是生成的ssh-key的名称，并强制要求为某个邮箱，甚至可以不是邮箱。那为什么使用邮箱来标识一个ssh-key？

    因为github账号或者gitee，gitlab都是使用邮箱来注册的，你可能不只有一个这类的账号，所以使用自己的邮箱账号可以知道这个ssh-key是由哪一个账号生成的。

  • 这里的密钥文件名也可也按照自己的喜好命名：github代表这个ssh是一个github的ssh-key，id_rsa是因为默认生成的ssh-key的文件名就是id_密钥类型，所以使用github_id_rsa。

  • -t 选择生成密钥的类型：rsa、 ed25519、dsa等

  • -C 指定此密钥的备注信息，一般推荐使用自己的账号的邮箱作为标识

  • -f 指定密钥目录位置， 默认为当前用户home路径下的 .ssh 隐藏目录, 也就是 ~/.ssh/目录下 , 同时默认密钥文件名以 id_密钥类型 开头.

    如果是 root 用户, 则在 /root/.ssh/目录下 , 若为其他用户, 则在/home/username/.ssh/目录下;

查看ssh-key

查看ssh-key公钥，使用命令 cat ~/.ssh/密钥文件名.pub，.pub代表该密钥是一个ssh-key公钥。

查看ssh-key私钥，使用命令cat ~/.ssh/密钥文件名
